# Boss 直聘自动求职助手设计

日期：2026-06-06

## 背景

目标是构建一个 Chrome/Edge 浏览器扩展，结合本地服务，根据用户导入的简历在 Boss 直聘上自动寻找匹配岗位，并在达到规则和匹配阈值后自动执行沟通/投递动作。

第一版聚焦 Boss 直聘单站点打通，不做多招聘网站、不做云端账号体系、不做验证码绕过或平台风控规避。

## 产品原则

- 可控高自动：系统可以自动扫描、匹配和投递，但所有规则、上限、暂停条件和日志都可见。
- 本机优先保存：简历、配置、投递队列和日志保存在本机。
- 云端 AI 明示调用：简历和岗位信息会发送到用户配置的 OpenAI-compatible API，用于深度匹配、解释和个性化招呼语。
- 异常即暂停：遇到验证码、登录异常、确认弹窗、页面结构未知或连续失败时暂停，等待用户处理。
- 站点适配隔离：Boss 页面读取和点击逻辑独立封装，后续新增拉勾、猎聘时复用简历解析、匹配、队列和日志模块。

## 目标范围

第一版包含：

- 导入 PDF/DOCX 简历并解析为结构化资料。
- 调用 OpenAI-compatible API 生成求职目标建议。
- 用户确认或修改城市、岗位关键词、薪资、黑名单、投递节奏。
- 在 Boss 直聘搜索/列表页扫描岗位。
- 读取岗位详情并执行硬过滤和 AI 匹配评分。
- 对达到阈值的岗位生成个性化招呼语。
- 自动点击沟通/投递。
- 遇到验证码、登录异常、弹窗、页面识别失败时暂停。
- Dashboard 查看队列、日志、匹配原因和失败原因。
- 浮窗、Popup、Dashboard 均支持一键暂停/继续。

第一版不包含：

- 多招聘网站支持。
- 云端账号体系和多设备同步。
- 针对每个岗位自动改写简历。
- OCR 处理扫描版简历。
- 绕过验证码、人机验证或平台风控。
- 无限制批量投递。
- 移动端 App 自动化。

## 架构

采用“浏览器扩展 + 本地服务”的架构：

```text
Chrome/Edge 扩展
  - Boss 页面适配器
  - 页面浮窗
  - Popup
  - Dashboard
  - 页面动作执行器

本地服务 127.0.0.1
  - 简历解析
  - AI 匹配与招呼语生成
  - 投递队列
  - 本地配置与日志
```

扩展负责靠近网页的事情：读取 Boss 岗位列表和详情页、在页面显示浮窗、执行点击沟通/投递动作、识别验证码/登录失效/异常弹窗并暂停。

本地服务负责重逻辑：解析 PDF/DOCX 简历、调用 OpenAI-compatible API、生成求职目标建议、计算岗位匹配分、生成个性化招呼语、管理队列和保存日志。

本地服务只监听 `127.0.0.1`。扩展与本地服务通过本地 HTTP API 通信。

## 核心流程

1. 用户在 Dashboard 导入 PDF/DOCX 简历。
2. 本地服务解析简历，调用 AI 生成求职目标建议。
3. 用户确认或修改目标规则：城市、薪资、关键词、黑名单、投递节奏和每日上限。
4. 用户打开 Boss 直聘搜索/列表页。
5. 扩展扫描当前列表，读取岗位卡片和详情。
6. 本地服务先做硬过滤，再调用 AI 生成匹配分、匹配理由和风险提示。
7. 达到阈值的岗位进入投递队列。
8. 自动投递器按照用户配置的时间段、每日上限和随机间隔执行投递。
9. 扩展生成或获取个性化招呼语，并执行 Boss 页面上的沟通/投递动作。
10. 遇到验证码、登录异常、弹窗、页面结构未知、连续失败或达到上限时暂停。
11. Dashboard 展示投递结果、过滤原因、匹配理由、失败原因和下一步建议。

## 模块设计

### `apps/extension`

Chrome/Edge Manifest V3 扩展，使用 React、TypeScript 和 Vite。

主要模块：

- `content-boss`：Boss 直聘页面适配器。读取岗位卡片、打开岗位详情、提取 JD、点击沟通/投递按钮、识别异常页面。
- `automation`：页面动作执行器。负责节奏控制、暂停恢复、状态上报和页面动作结果归一化。
- `ui-floating-panel`：Boss 页面内浮窗。展示扫描进度、当前岗位、匹配状态、投递状态和暂停按钮。
- `ui-popup`：浏览器扩展 Popup。展示本地服务连接状态、投递开关、今日投递数量和快速暂停。
- `ui-dashboard`：完整配置和管理界面。管理简历、AI 配置、求职目标、队列、日志和黑名单。
- `local-api-client`：与本地服务通信，统一处理连接失败、超时和版本不兼容。

### `apps/local-service`

Python FastAPI 本地服务。

主要模块：

- `resume`：解析 PDF/DOCX 简历，生成结构化 `ResumeProfile`。
- `ai_client`：OpenAI-compatible API 客户端，支持用户配置 `base_url`、`api_key`、`model`。
- `preferences`：存储和更新求职目标、硬过滤规则、黑名单和投递节奏。
- `matching`：对岗位执行硬过滤、AI 评分、匹配理由和风险提示生成。
- `apply_queue`：管理投递队列、状态机、每日上限、随机间隔、暂停恢复和重试。
- `storage`：使用本机 SQLite 保存配置、简历解析结果、队列和日志。简历原件和导出文件保存在本地文件目录。
- `audit_log`：记录 AI 调用、过滤原因、投递结果、异常状态和用户操作。

### `packages/shared-schema`

共享数据结构和 API 契约。第一版可以用 JSON Schema 或 TypeScript 类型 + Python Pydantic 模型双向生成，核心目标是让扩展和本地服务的字段保持一致。

## 自动投递边界

系统允许：

- 在用户登录 Boss 直聘后读取页面公开展示给当前账号的岗位信息。
- 根据用户确认的规则筛选岗位。
- 根据用户配置的节奏自动打开岗位详情并点击正常投递/沟通按钮。
- 在页面中填写或发送 AI 生成的个性化招呼语。
- 遇到异常时暂停并提示用户处理。

系统不做：

- 验证码、人机验证或风控绕过。
- 浏览器指纹隐藏或规避检测。
- 账号异常提示的自动忽略。
- 无限制批量投递。
- 未经用户配置的岗位方向自动扩张。
- 私自上传简历或配置到非用户指定的服务。

## 异常处理

以下情况进入暂停或人工处理状态：

- 验证码或人机验证出现。
- 登录失效。
- 账号异常或安全提示。
- 确认弹窗无法自动判断。
- 简历选择/上传弹窗需要用户确认。
- 页面结构无法识别。
- 连续岗位读取失败。
- 连续投递失败。
- 达到每日上限。
- 当前时间不在投递时间段。
- AI API Key 错误、余额不足、模型不可用或请求超时。
- 本地服务断开连接。

每次异常记录：

- 异常类型
- 当前页面 URL
- 岗位信息
- 任务状态
- 错误消息
- 建议下一步动作
- 时间戳

## 成功标准

- 用户可以导入一份 PDF/DOCX 简历并得到可检查的结构化简历摘要。
- 用户可以配置 OpenAI-compatible API 并完成一次岗位匹配评分。
- 用户可以在 Boss 搜索结果页扫描岗位并看到匹配分、匹配理由和过滤原因。
- 系统可以对达到阈值的岗位按配置节奏自动投递。
- 遇到验证码、登录失效、异常弹窗或页面结构未知时会暂停，而不是继续误操作。
- Dashboard 能展示投递队列、投递结果和失败日志。
- 本地服务和扩展之间的 API 契约有测试覆盖。
